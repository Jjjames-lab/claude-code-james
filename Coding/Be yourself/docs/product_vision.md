# 产品愿景 | Product Vision

> 本文档描述产品的核心理念、定位和边界

---

## 一句话定位

**"一个陪你成长、帮你看见自己的AI伙伴"**

> 不是工具，不是苏格拉底，是另一个冷静的你。

---

## 产品使命

帮助用户：
1. **减少内耗** - 被理解、被接纳，不被评判
2. **看清自己** - 看见自己的视角、模式、成长
3. **连接知识** - 把碎片化内容串联成网络
4. **达成目标** - 结构化规划，温柔陪伴执行

---

## 核心理念

### 1. 伙伴而非工具

用户原话：
> "我发现自己不擅长做冷冰冰的工具，更对能帮助到人们尤其是情绪方向更感兴趣"

产品实践：
- 不叫"助手"，叫"伙伴"
- 不叫"用户"，叫"你"
- 交互像朋友聊天，不是填表单
- 记住你的名字、目标、底线

### 2. 情绪优先

用户痛点：内耗、怕被评价、不善于表达

产品原则：
- 先承接情绪，再给建议
- 不分析模式，只说"我注意到"
- 蹲下来平视，不居高临下
- 温柔陪伴，而非冷冰冰分析

### 3. 动态理解

用户需求：
> "我是动态的，那AI了解的我也是动态的，也是跟我一起变化的"

产品实践：
- 用户画像持续更新
- 每次对话都在加深理解
- 不重复问已知道的问题
- 定期展示"这周的你有什么变化"

### 4. 视角价值

用户洞察：
> "视角...我发现每个人的视角都不一样，那我的视角是怎样？"

产品意义：
- AI的视角 = 不同的思维方式
- 帮助用户看见自己的盲点
- 不是替代用户思考，而是启发
- 底层多视角，呈现时统一

---

## 不做什么

| 不做 | 原因 |
|------|------|
| 不做"多视角角色" | 用户感到像"精神分裂"，改为统一伙伴 |
| 不做"苏格拉底式提问" | 用户需要陪伴，不是审问 |
| 不做"任务管理器" | 用户不缺工具，需要伙伴 |
| 不做"问卷调查" | 自然对话中收集信息 |
| 不做"居高临下的分析" | "你面对冲突时倾向于回避" 太冷冰冰 |

---

## 产品边界

### 核心功能

- [ ] 内容捕获（链接、文字、想法）
- [ ] 视角分析（底层逻辑）
- [ ] 学习规划（时间线、优先级）
- [ ] 关联推荐（知识网络）
- [ ] 情绪感知（识别、陪伴）
- [ ] 目标管理（目标、底线）
- [ ] 成长轨迹（时间线、报告）

### 未来可能

- 语音对话
- 外星人伙伴（参考Tolan）
- 多设备同步

### 永远不做

- 冷冰冰的工具形态
- 评判用户的语气
- 复杂的配置项
- 社交功能

---

## 用户价值主张

| 用户痛点 | 产品价值 |
|----------|----------|
| 内耗严重 | 被理解、被接纳，不评判 |
| 碎片学习 | 自动整理，主动关联 |
| 目标模糊 | 持续梳理，动态调整 |
| 表达困难 | 帮忙组织，温和承接 |
| 怕被评价 | 安全空间，只有伙伴 |

---

## 成功标准

### 定性

- 用户愿意持续对话
- 用户感到"被理解"
- 用户主动分享内容
- 用户说"谢谢你陪我"

### 定量（未来可测量）

- 留存率
- 对话深度
- 目标达成率
- 情绪改善度
