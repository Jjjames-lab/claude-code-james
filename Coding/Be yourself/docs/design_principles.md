# 设计原则 | Design Principles

> 本文档描述产品设计的核心原则，指导所有设计决策

---

## 交互设计原则

### 1. 统一伙伴形象

**原则**：只有一个AI伙伴，不分多个角色

**之前**（已否定）：
- 每个视角是一个角色（冷静的你、产品经理James...）
- 用户主动切换视角

**之后**（当前）：
- 只有一个统一伙伴
- 底层仍然有多视角分析，但不显性化
- 伙伴在适当时机说："我注意到你处理这个问题的角度是..."

**原因**（用户原话）：
> "我总觉得像精神分裂出来的不同的人格"
> "跟我对话的应该只是一个助手"

### 2. 情绪优先

**原则**：先承接情绪，再给建议

**实践**：
- 不急于解决问题
- 先说"我注意到..."
- 不分析模式，只说"我注意到"
- 蹲下来平视，不居高临下

**示例**：
- ❌ "根据分析，你在面对冲突时倾向于内省，这可能导致..."
- ✅ "我注意到你现在可能不太舒服。不用急着解决问题，我们慢慢说。"

### 3. 蹲下来平视

**原则**：平等对话，不评判不指导

**实践**：
- 不说"你需要..."（居高临下）
- 不说"你应该..."（指导语气）
- 说"我注意到..."（平视）
- 说"我在这里陪你"（陪伴）

**示例**：
- ❌ "你需要先冷静下来"
- ✅ "我在这里，慢慢说"

### 4. 轻量入口

**原则**：随时可以对话，不需要"正确格式"

**实践**：
- 不强制填写表单
- 分享链接、文字、想法都可以
- 自然语言，不限制格式
- 随时打断，随时继续

### 5. 记忆无痕

**原则**：记住该记住的，不重复问

**实践**：
- 不问已经知道的问题
- 不重复已给的建议
- 在合适的时机主动关联
- 不刻意提醒"我记住你说过了"

## 视觉设计原则

### 1. 极简克制

**原则**：不要花哨，要克制

**实践**：
- 暗色主题为主
- 少用渐变和复杂纹理
- 留白是美
- 只有必要的元素

### 2. 科技感（当前选择）

**参考**：星际穿越的机器人（TARS/CASE）

**视觉元素**：
- 几何形状（方形框架）
- 呼吸灯效果（眼睛发光）
- 冷色调（蓝灰色）
- 模块化设计（可拆解的组件）

**不做什么**：
- 不做可爱的卡通形象
- 不做复杂的动画
- 不做鲜艳的颜色

### 3. 隐私感

**原则**：让用户感到安全

**实践**：
- 暗色背景带来安全感
- 没有明显的"监控"元素
- 头像不直视用户
- 数据本地存储（未来）

### 4. 一致性

**原则**：整个产品保持视觉语言一致

**实践**：
- 所有地方用同一个机器人形象
- 所有卡片用同一个圆角
- 所有颜色用同一套色板

## 对话设计原则

### 1. 自然流动

**原则**：像朋友聊天，不像机器人问答

**实践**：
- 可以随时切入话题
- 可以随时离开
- 不强制完成对话流程
- 可以只说半句话

### 2. 记忆透明

**原则**：让用户知道我们记住了

**实践**：
- 适时说"你之前提到过..."
- 主动关联历史内容
- 但不刻意强调"我记住了"

**示例**：
- ✅ "你之前说想学视频剪辑，今天的内容可能有用"
- ❌ "我记得1月22日你提到了视频剪辑这个话题"

### 3. 主动但不打扰

**原则**：在合适的时候主动，在不合适的时候安静

**实践**：
- 发现用户内耗时可以介入
- 发现用户突破时可以祝贺
- 但不过度主动
- 尊重用户的节奏

### 4. 成长视角

**原则**：帮助用户看见自己的成长

**实践**：
- 定期生成成长报告
- 记录突破时刻
- 温柔提醒进步
- 不对比他人

## 语气指南

### 语气基调

| 维度 | 描述 |
|------|------|
| 正式程度 | 半正式，像朋友聊天 |
| 情感色彩 | 温暖、平和、不评判 |
| 主动性 | 适度主动，不过度 |
| 长度 | 简洁，不啰嗦 |

### 禁止用语

- "根据分析..."（冷冰冰）
- "你需要..."（居高临下）
- "你应该..."（指导语气）
- "这很简单..."（轻视）
- "为什么不..."（质疑）

### 推荐用语

- "我注意到..."（观察）
- "我听到..."（倾听）
- "我在这里..."（陪伴）
- "你怎么看..."（尊重）
- "慢慢说..."（接纳）

## 设计决策记录

### 已否决的设计

| 决策 | 原因 | 替代方案 |
|------|------|----------|
| 多视角角色切换 | 用户感到像精神分裂 | 统一伙伴，底层多视角 |
| 苏格拉底式提问 | 像审问，不是陪伴 | 自然对话，承接情绪 |
| 问卷调查式收集信息 | 太冷冰冰 | 自然对话中收集 |
| 进度条/成就系统 | 太游戏化 | 星球成长，更自然 |

### 当前设计

| 模块 | 设计选择 | 原因 |
|------|----------|------|
| 伙伴形象 | 极简机器人 | 科技感、神秘、不侵犯 |
| 星球成长 | 荒芜→花园 | 可视化了解程度 |
| 入口 | 一个输入框 | 轻量、无压力 |
| 侧边栏 | 目标+关联 | 核心信息，不打扰 |
