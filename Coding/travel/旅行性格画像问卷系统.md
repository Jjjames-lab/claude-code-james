# 旅行性格画像问卷系统

**基于7个核心要素设计**
**目标**：通过10-12个问题，精准描绘用户的旅行性格画像

---

## 设计原则（7个核心要素）

### 1. 共情能力 - 理解情绪背后的原因
- 不只是问"你喜欢什么"，而是理解"为什么喜欢"
- 通过场景化问题，让用户感受到"你懂我"

### 2. 被接受 - 不评判、无标准答案
- 强调"没有对错，只有不同"
- 避免引导性问题
- 让每个选择都被尊重

### 3. 主动性 - 提供相关示例引导
- 给出具体场景和例子
- 帮助用户回忆和思考
- 降低回答门槛

### 4. 安全感 - 信息真实可靠
- 问题设计专业、有逻辑
- 避免过于隐私的问题
- 建立信任感

### 5. 问题设计 - 该直接直接，该场景化场景化
- 硬性信息（预算、时间）直接问
- 软性偏好（感受、价值观）场景化
- 混合使用不同题型

### 6. 即时反馈 - 让人有"被看见"的感觉
- 问卷过程中给出即时反馈
- 让用户感受到"你在认真听我说"
- 创造对话感

### 7. 允许复杂 - 接受矛盾和多面性
- 承认人是复杂的，可以有矛盾的选择
- 允许"视情况而定"的选项
- 不把人简单标签化

---

## 系统设计底层思路与思想

### 核心洞察

**人不只是复杂的，还是动态的、情境依赖的。**

传统的问卷系统往往陷入两个误区：
1. **标签化** - 用ABCD把人分类，但人是多维的
2. **静态化** - 假设人的偏好是固定的，但实际上偏好会随情境变化

我们的系统要解决这两个问题。

---

### 设计思想一：情境依赖性

**问题**：同一个人的选择会因**旅行伙伴、旅行目的、身体状态**而不同。

**举例**：
- 和父母去 vs 和朋友去 → 计划度、社交偏好完全不同
- 度假放松 vs 朝圣打卡 → 体验深度不同
- 精力充沛 vs 疲惫期 → 探索欲不同

**解决方案**：
在问卷中增加"情境校准"问题，了解用户**当前这次旅行**的状态和期望，而不是只依赖固定的"性格画像"。

**新增问题示例**：
```
这次旅行，你希望：
A. 完全放松，不用动脑子
B. 正常节奏，有计划但不累
C. 充实探索，充分利用时间
```

**价值**：根据用户**当前状态**调整推荐，而不是用一个固化的"性格"套所有情况。

---

### 设计思想二：矛盾合理化

**核心问题**：虽然系统说"允许复杂"，但传统的评分系统还是会把人往一个方向推。

**举例**：
- "高计划度"通常被理解为"不喜欢意外"
- "高探索欲"通常被理解为"不喜欢计划"
- 但现实中，有人既是"高计划度"又是"高探索欲" —— 他们可能计划了很多探索项目

**传统评分系统的问题**：
- 用线性分数表示每个维度（1-5分）
- 但没有解释**维度之间的关系**
- 用户看到"计划度5分"，会以为"所以你不接受意外？"

**解决方案**：

1. **在画像输出时，展示组合的合理性**
   ```
   📋 计划度：⭐⭐⭐⭐⭐ + 🌍 探索欲：⭐⭐⭐⭐⭐
   → 你是个"有准备的探索者"，喜欢把惊喜也纳入计划！
   ```

2. **增加"矛盾说明"机制**
   - 检测维度之间的特殊组合
   - 给出合理的解释，而不是让用户自己困惑
   - 例如："高计划度 + 高意外接受度 = 有备而来的惊喜"

3. **避免"分数越xx越好"的暗示**
   - 5分不代表"好"，1分不代表"差"
   - 每个分数都有其合理性和场景

**技术实现要点**：
- 需要定义维度组合的解释规则
- 不能只看单个维度，要看**维度关系**
- 画像输出要强调"组合的合理性"，而不是"每个维度的高低"

---

### 设计思想三：旅行经验维度

**问题**：新手和老手的旅行方式完全不同，但这6个维度没有体现。

**举例**：
- 新手说"我喜欢计划" → 可能是因为不敢随意
- 老手说"我喜欢随性" → 是因为有经验所以自信
- 同样的"高计划度"，背后的含义完全不同

**解决方案**：

增加"旅行经验"维度，作为**解读其他维度的背景**。

**新增问题**：
```
你之前的旅行经验如何？
A. 几乎没旅行过，这是我的第一次/第二次
B. 有一些旅行经验，去过5-10个地方
C. 旅行达人，经常出去
```

**价值**：
- 新手需要更多引导和安全感
- 老手需要更多小众推荐
- 同样的"高计划度"，对新手的解读是"需要详细攻略"，对老手的解读是"高效利用时间"

---

### 设计思想四：动态画像

**核心问题**：现在的画像看起来是静态的，但人是会变的。

**表现**：
- 一次旅行后，人的偏好可能改变
- 不同年龄段、不同人生阶段，旅行方式不同
- 固定的画像会过时

**解决方案**：

1. **在结束语中埋下"动态"的种子**
   ```
   人的旅行方式会随时间和经历变化，
   下次旅行时，欢迎再来做这个问卷，
   看看你的旅行性格有没有新的变化～
   ```

2. **允许用户修正画像**
   - 生成行程后，问"这个行程适合你吗？"
   - 如果用户说"太累了"，系统学习到"我高估了你的计划度"
   - 下次推荐时自动调整

3. **技术上的持续学习**
   - 记录用户的反馈
   - 用反馈数据微调画像
   - 但要明确告知用户数据用途

**哲学意义**：
- 不是"给你贴一个标签"
- 而是"帮助我们更好地理解你"
- 允许成长和变化

---

### 设计思想五：深度共情

**问题**：开放性问题的AI分析，不能只是"提取信息"，而要"共情"。

**举例**：

用户回答："在清迈和一位手艺人聊天，买了他的作品"

**不好的分析**：
- ❌ 用户喜欢文化
- ❌ 用户喜欢艺术
- ❌ 用户喜欢购物

**好的分析**：
- ✅ 你似乎很珍视那些"有温度的连接时刻"
- ✅ 比起宏大的景点，你更在意人与人之间的真诚交流
- ✅ 你愿意为有意义的体验付费（买作品）

**技术实现要点**：

使用 Claude API 分析时，不只是提取关键词，而是：
1. **识别情感** - 开心、满足、感动、兴奋？
2. **识别价值观** - 这件事对用户意味着什么？
3. **识别动机** - 为什么这件事让他难忘？
4. **生成有温度的反馈** - 让用户感受到"你真的懂我"

**示例反馈**：
```
听起来你是个注重深度体验的人，
那些有温度的连接时刻对你来说特别有意义。
比起打卡景点，你更在意和人之间的真诚交流，
这真的很珍贵～
```

---

### 技术实现的关键原则

基于以上设计思想，技术实现时要遵循：

1. **不只是记录答案，还要理解关系**
   - 维度之间的组合关系
   - 答案背后的动机
   - 情境对选择的影响

2. **不只是生成画像，还要持续学习**
   - 记录用户反馈
   - 允许修正
   - 动态调整

3. **不只是分析数据，还要共情**
   - AI分析要有温度
   - 反馈要让人感到被理解
   - 避免"机器感"

4. **不只是输出结果，还要解释合理性**
   - 解释为什么这样推荐
   - 解释矛盾组合的合理性
   - 让用户信服

## 第一部分：旅行场景偏好（场景化问题）

### 问题 1：目的地的选择
**类型**：场景描述 + 单选

**题目**：
```
假设你有一个7天假期，你会更倾向于：

A. 提前做足功课，列好必打卡清单，把每天安排得满满当当
   （比如：早上去某家网红咖啡店，中午打卡米其林小吃，
     下午逛艺术市集，晚上去特色酒吧）

B. 大致确定2-3个想去的点，其他时间随性探索
   （比如：今天去古城，但具体逛哪里、吃什么，到了再看心情）

C. 完全不计划，到了再说，走到哪算哪
   （比如：起床再看天气，决定今天做什么）
```

**考察维度**：计划度
**即时反馈**：
```
选择A：你是个有计划的探索者，喜欢把时间充分利用起来！
选择B：你懂得平衡，既有方向又保持灵活性～
选择C：你追求完全的自由，享受意外发现的惊喜！
```

---

## 第二部分：价值观与偏好（认同度打分）

```
1-2分：你更享受轻松的旅行，不需要太多意义赋予
```

## 画像生成逻辑

### 维度评分计算

#### 1. 探索欲（保守-冒险）
- 问题2：A=5分，B=3分，C=1分
- 问题7：1-5分
- 问题8：C、F选项加分

#### 2. 计划度（随性-计划）
- 问题1：A=5分，B=3分，C=1分
- 问题7：1-5分

#### 3. 舒适区（熟悉-新奇）
- 问题2：A=5分，B=3分，C=1分
- 问题8：C、D、F选项加分

#### 4. 社交偏好（独处-社交）
- 问题3：A=2分，B=3分，C=5分
- 问题9：E选项加分

#### 5. 体验深度（打卡-沉浸）
