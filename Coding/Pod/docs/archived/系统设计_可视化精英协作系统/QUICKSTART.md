# 快速开始指南 - 可视化精英协作系统

> **目标**: 5分钟内上手使用

---

## 🚀 立即体验

### Step 1: 打开看板（10秒）

```bash
cd "/Users/tbingy/Desktop/Claude Code/Coding/Pod/系统设计_可视化精英协作系统"
open demo.html
```

**你会看到**：
- 5个角色的状态卡片
- 项目整体进度
- 最近事件流
- 任务队列

### Step 2: 观察自动刷新（30秒）

看板会每30秒自动刷新一次，状态栏的时间会更新。

### Step 3: 尝试更新状态（2分钟）

打开更新界面：
```bash
open update.html
```

填写表单：
1. 选择角色（例如：架构师）
2. 点击"🟡 工作中"
3. 填写任务名称（例如：多引擎架构设计）
4. 拖动进度条到60%
5. 填写已用时间（例如：15分钟）
6. 填写预计时间（例如：35分钟）
7. 填写事件描述（例如：完成接口设计初稿）
8. 点击"💾 更新状态"

浏览器会下载一个新的 `state.json` 文件。

### Step 4: 查看更新效果（10秒）

将下载的 `state.json` 替换原文件：
```bash
# 备份原文件
cp state.json state.json.backup

# 替换为新文件（将下载的文件移动到当前目录）
# 在Finder中，将下载的state.json移动到 系统设计_可视化精英协作系统/ 文件夹
```

刷新 `demo.html`，你会看到架构师的状态已更新。

## 📖 核心概念

### 状态 (Status)

- 🟢 **空闲**: 可以分配任务
- 🟡 **工作中**: 正在执行任务
- 🔴 **阻塞**: 遇到问题，需要帮助

### 事件流 (Event Stream)

按时间倒序展示的系统事件：
- ✅ 任务完成
- 🟡 任务开始
- 🚨 警告/冲突
- 🔓 阻塞解除
- 🚀 系统启动

### 任务队列 (Task Queue)

展示所有任务的状态：
- 进行中
- 等待中（pending）
- 依赖关系

## 🎯 使用场景

### 场景1: 你想了解全局进展

**操作**：打开 `demo.html`

**信息获取**：
- 项目整体进度多少？
- 谁在空闲，可以分配任务？
- 有没有角色被阻塞？
- 最近发生了什么？

### 场景2: 角色想更新进度

**操作**：
1. 打开 `update.html`
2. 填写表单
3. 提交更新
4. 下载新的 `state.json`
5. 替换原文件

**注意**：MVP版本需要手动替换文件，V2会自动化

### 场景3: 你想查看历史事件

**操作**：在 `demo.html` 中滚动到"📋 实时事件流"部分

**信息获取**：
- 后端什么时候完成的ASR研究？
- 架构师什么时候开始的设计？
- 有没有冲突发生？

### 场景4: 你发现问题需要干预

**操作**：
1. 在 `demo.html` 中看到 🔴 阻塞状态
2. 了解阻塞原因（查看needs或blocking字段）
3. 通知相关角色
4. 解决后角色更新状态为 🟢

## 💡 最佳实践

### 给监控者（你）

1. **定期查看**：建议每隔1-2小时打开看板
2. **主动发现问题**：不要等角色汇报
3. **及时干预**：发现阻塞立即处理
4. **培养质疑文化**：鼓励角色提出更好的方案

### 给各角色

1. **工作前**：查看看板，了解全局
2. **工作中**：每完成25%进度更新一次
3. **遇到问题**：立即标记为 🔴 阻塞
4. **完成任务**：更新为 🟢 空闲，添加完成事件

## ⚠️ 常见问题

### Q1: 为什么更新后需要下载文件？

**A**: 这是 MVP 版本的限制。浏览器不能直接写入文件系统，所以需要下载并手动替换。

**解决方案**：
- 现阶段：手动替换文件
- V2 版本：会提供本地服务器，自动写入

### Q2: 30秒刷新是不是太慢？

**A**: 对于 MVP 来说已经足够。如果你需要更实时的更新：

**临时方案**：手动刷新浏览器
**长期方案**：V2 会提供 Server-Sent Events，真正实时

### Q3: 如何查看更多历史事件？

**A**: 当前版本只显示最近10条事件。

**解决方案**：
- 查看 `state.json` 中的 `events` 数组
- V2 会添加历史事件查看功能

### Q4: 可以自动通知下一个角色吗？

**A**: 当前版本不能，这是 MVP 的限制。

**V2 计划**：
- 任务完成 → 自动通知下一个角色
- 检测到阻塞 → 自动通知你
- 发现冲突 → 自动创建 Challenge
