# 章节区域优化方案

## 当前问题
```
┌─────────────────────────────────┐
│ [1] [0:00]                      │  ← 序号+时间戳
│                                 │
│ 节目介绍                        │  ← 标题
│ Program Introduction            │  ← 翻译
│                                 │
│ • 索尼耳机冠名                  │  ← 要点原文
│   Sony Headphones Sponsored     │  ← 要点翻译
│ • 生动早咖啡节目                │
│   Vivid Early Morning Coffee... │  ← 每个要点2行
│ • 零食连锁...                   │
│   Snack Chain...                │
│ +1 更多                         │
└─────────────────────────────────┘
```

**拥挤原因**：
- 双语模式下每个要点占2行
- padding-5 (20px) 内边距
- gap-12px 要点间距
- 序号+时间戳占用顶部空间

---

## 方案1：极简折叠版 ⭐推荐

### 设计理念
"默认简洁，按需展开" - 只显示核心信息，要点默认隐藏

### 视觉效果
```
┌─────────────────────────────────┐
│ 1 0:00  节目介绍                │  ← 单行紧凑布局
│         Program Introduction    │  ← 翻译（小字）
└─────────────────────────────────┘
         ↓ 点击展开
┌─────────────────────────────────┐
│ 1 0:00  节目介绍                │
│         Program Introduction    │
│                                 │
│ • 索尼耳机冠名 / Sony...        │  ← 要点内联显示
│ • 生动早咖啡 / Vivid...         │  ← 原文/翻译同行
└─────────────────────────────────┘
```

### 优点
- ✅ 极致简洁，默认只显示标题
- ✅ 可快速浏览所有章节
- ✅ 需要时才展开查看要点
- ✅ 移动端友好

### 缺点
- ⚠️ 需要额外点击才能看到要点
- ⚠️ 不能一眼看到所有信息

### 实现细节
- 移除卡片padding，改用紧凑布局
- 序号+时间戳+标题单行显示
- 翻译小字显示在标题下方
- 默认折叠要点，点击展开
- 要点用斜杠分隔：`原文 / Translation`

---

## 方案2：内联优化版

### 设计理念
"保持信息，优化排版" - 所有信息都显示，但优化空间利用

### 视觉效果
```
┌─────────────────────────────────┐
│ 1  0:00                         │  ← 序号+时间更小
│                                 │
│ 节目介绍                        │
│ Program Introduction            │
│                                 │
│ • 索尼耳机冠名 / Sony Head...   │  ← 要点内联
│ • 生动早咖啡节目 / Vivid E...   │  ← 同行显示
│ • 零食连锁上市 / Snack Chain... │
└─────────────────────────────────┘
```

### 优点
- ✅ 保留所有信息
- ✅ 垂直空间减少50%
- ✅ 不需要额外点击

### 缺点
- ⚠️ 要点文字可能被截断
- ⚠️ 长翻译可能导致换行
- ⚠️ 视觉层次不够清晰

### 实现细节
- 减小padding: p-5 → p-4
- 减小间距: gap-12px → gap-8px
- 序号缩小: 7×7 → 6×6
- 要点原文+翻译同行，用斜杠分隔
- 文字溢出省略号

---

## 方案3：混合版（渐进显示）

### 设计理念
"核心先行，次要后显" - 标题+第一个要点默认显示，其余折叠

### 视觉效果
```
┌─────────────────────────────────┐
│ 1  0:00                         │
│ 节目介绍 / Program Introduction  │  ← 标题内联
│                                 │
│ • 索尼耳机冠名 / Sony...        │  ← 第一个要点
│                                 │
│ [+2 更多要点]                   │  ← 折叠提示
└─────────────────────────────────┘
         ↓ 点击展开
┌─────────────────────────────────┐
│ ...                            │
│ • 索尼耳机冠名 / Sony...        │
│ • 生动早咖啡 / Vivid...         │
│ • 零食连锁上市 / Snack Chain... │  ← 展开所有
└─────────────────────────────────┘
```

### 优点
- ✅ 平衡简洁和信息
- ✅ 默认显示第一个要点作为预览
- ✅ 可以快速判断是否需要展开

### 缺点
- ⚠️ 第一个要点的选择可能不够智能
- ⚠️ 仍然需要点击展开

### 实现细节
- 标题原文+翻译内联显示
- 默认只显示第一个要点
- 其余要点折叠为"+N 更多"
- 点击展开所有要点

## 推荐方案

**我推荐：方案1（极简折叠版）**

### 理由
1. **符合产品哲学**："慢下来，深思考" - 不需要一次性看到所有信息
2. **信息层次清晰**：标题→概览→详情，符合认知规律
3. **空间效率最高**：默认状态只占1/3空间
4. **可扩展性好**：未来可以加"全部展开"按钮

### 实现优先级
1. **第一阶段**：实现方案1的核心功能（折叠/展开）
2. **第二阶段**：优化动画效果
3. **第三阶段**：添加"全部展开/收起"快捷操作
