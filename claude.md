## 📋 必须遵守的开发规范

### 1. 第三方API集成流程
```
✅ 正确：
1. 先问用户: "你有官方demo代码或API文档吗？"
2. 如果有 → 基于官方代码实现
3. 如果没有 → 我搜索文档，但要明确告知用户文档来源
4. 实施前 → 向用户确认方案
5. 用户批准 → 开始实施

❌ 禁止：
- 不索要官方资料，自己搜索
- 使用非官方、过时的文档
- 不向用户确认直接实施
```

### 2. 前后端接口开发流程
```
✅ 正确：
1. types.ts → 定义接口（TypeScript）
2. 后端 → 按接口实现（Pydantic）
3. 前端 → 使用类型定义
4. 测试 → 打印实际返回，对比接口

❌ 禁止：
- 先写后端，后写前端
- 后端返回随意格式
- 前端"猜测"返回格式
```

### 3. 数据架构设计原则
```
✅ 正确：
- 后端: 数据转换（HTML → JSON）
- 前端: UI渲染（基于JSON）

❌ 禁止：
- 后端返回原始HTML
- 前端显示HTML源码
- 让前端解析复杂格式
```

### 4. 状态管理决策树
```
问自己3个问题：
Q1: 这个数据会被多个组件共享吗？
Q2: 这个数据需要在组件卸载后保留吗？
Q3: 这个数据是全局唯一的吗？

任意YES → 父组件状态或Store
全部NO → 组件内部状态
```
